# LEVEL UP - Technical Architecture & Project Documentation

This document provides a comprehensive technical overview of the **LEVEL UP** ecosystem. It serves as the primary reference for the project's architecture, design patterns, and core functionalities, suitable for project reports and developer onboarding.

---

## 1. Project Overview
**LEVEL UP** is a state-of-the-art productivity and educational platform designed for the modern student. It integrates AI-driven analytics, real-time focus monitoring, and intelligent study tools to optimize the learning process.

### Key Value Propositions
*   **AI-Powered Accountability:** Uses facial detection to ensure the student remains focused during study sessions.
*   **Data-Driven Insights:** Visualizes study patterns using advanced charting libraries.
*   **Integrated AI Tools:** Summarizes complex PDFs and generates customized learning paths.
*   **Holistic Ecosystem:** Includes scheduling, resource management, and parental oversight.

### Tech Stack
*   **Frontend:** React 19 (Vite-powered)
*   **Backend/Auth:** Supabase (Database, Authentication, Storage)
*   **Styling:** Tailwind CSS 4.0 + Custom CSS Design System
*   **AI/ML:** `face-api.js` (Browser-based Vision AI)
*   **Analytics:** Recharts (SVG-based charting)
*   **Data Handling:** Axios, Day.js
*   **Export/Reporting:** html2canvas, jsPDF

## 3. Core Architectural Modules

### A. Focus Engine (AI Presence Detection)
*   **Logic:** Implements a real-time monitoring loop using the `tinyFaceDetector` model from `face-api.js`.
*   **Features:** Automatic timer pausing when the user is absent, focus score calculation, and dynamic UI feedback.

### B. Dashboard Analyzer (Intelligence Hub)
*   **Logic:** Aggregates user logs into visual metrics.
*   **Visualizations:** Hourly activity (Bar), Section Usage (Pie), Focus Trends (Line), and Subject Coverage (Radar).
*   **Export:** Allows users to download a professional PDF recap of their daily productivity.

### C. AI Learning Path & Summarizer
*   **Logic:** Interfaces with AI backends (via webhooks or direct LLM integration) to process unstructured data (PDFs) into structured summaries and actionable learning roadmaps.

### D. Centralized State & Layout
*   **Design Pattern:** Uses a persistent Layout wrapper. Communication between components is handled via a mix of `localStorage` sync, custom `window.dispatchEvent` events, and Supabase real-time listeners.

## 4. Design Philosophy
*   **Aesthetics:** Dark-mode primary interface with "Cyber-Green" and "Deep-Purple" accents.
*   **Responsiveness:** Fully fluid layout using CSS Grid and Flexbox.
*   **Micro-interactions:** Subtle hover effects and transitions to enhance the premium feel.
